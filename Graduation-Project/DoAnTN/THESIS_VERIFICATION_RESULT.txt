USAGE DEMONSTRATIONS
================================================================================

Demo 1: Single File Test Generation
───────────────────────────────────

Command:
  mvn exec:java -Dexec.mainClass="com.doantn.app.App" \
    -Dexec.args="src/main/java/com/doantn/example/Calculator.java"

Input: src/main/java/com/doantn/example/Calculator.java
  - Calculator class with 6 public methods
  - Methods: add, subtract, multiply, divide, isPositive, getNumberType

Process:
  1. AnalyzerService parses Calculator.java
  2. Extracts: ClassModel with 6 MethodModel objects
  3. TestGenerator generates test files for each method
  4. JavaPoetTestGenerator creates JUnit 5 test code

Output: target/generated-tests/
  ✓ 6 test files generated (one per method)
  ✓ Each test file contains @Test annotated methods
  ✓ Assertions automatically generated
  ✓ Ready to run: mvn test


Demo 2: Directory Test Generation
──────────────────────────────────

Command:
  mvn exec:java -Dexec.mainClass="com.doantn.app.App" \
    -Dexec.args="src/main/java target/my-tests"

Input: src/main/java/ (all Java files)
  - Recursively scans directory
  - Finds all .java files
  - Analyzes each file

Process:
  1. AnalyzerService.analyzeDirectory() walks file tree
  2. For each .java file, creates ClassModel
  3. TestGenerator processes all ClassModels
  4. Generates comprehensive test suite

Output: target/my-tests/
  ✓ Test files for all classes
  ✓ Organized by package structure
  ✓ Complete test suite


Demo 3: Programmatic Usage
──────────────────────────

Code:
  AnalyzerService analyzer = new AnalyzerService();
  ClassModel model = analyzer.analyze("path/to/MyClass.java");

  TestGenerator generator = new TestGenerator("output-dir");
  generator.generateTests(model);

Usage:
  - Can be integrated into build process
  - Can be used in IDE plugins
  - Can be used in CI/CD pipelines
